# AWS Lightsail Deployment Notes

## Production startup

Use Gunicorn in production:

```bash
gunicorn --bind 0.0.0.0:${PORT:-8000} --workers 2 --threads 4 --timeout 120 wsgi:app
```

The current `Procfile` has been updated to match this command so the app can keep using the same WSGI entrypoint during migration.

## Environment variables to carry forward

- `EDGERANKED_NBA_BASE_DIR`
- `EDGERANKED_MLB_BASE_DIR`
- `EDGERANKED_UFC_BASE_DIR`
- `EDGERANKED_PGA_BASE_DIR`
- `PORT` for the web process

## Render-specific assumptions found

- `render.yaml` is still Render-specific and should not be used on Lightsail.
- `scripts/publish_render_site.sh` is named and scoped for the current Render snapshot flow.
- `scripts/publish_render_snapshot.py` now uses environment variables, but it is still Render-oriented by name and should be renamed or generalized during the actual migration.
- `nba_model/pipeline/service.py` still references `publish_render_site.sh`; that is fine for now, but it should be renamed or made provider-neutral during the actual migration.

## Linux path updates to plan for

- `nba_model/webapp/app.py` still falls back to `~/Desktop/pga_model` and `~/pga_model` for PGA data discovery. On Lightsail, set `EDGERANKED_PGA_BASE_DIR` explicitly.
- Runner shell scripts now resolve the repo root relative to their own location and fall back to `python3` if `.venv/bin/python` is unavailable.
- `scripts/publish_render_snapshot.py` now uses `EDGERANKED_MLB_SOURCE_DIR` first, then `EDGERANKED_MLB_BASE_DIR`, so MLB source location is Linux-safe and configurable.

## Waitlist storage

- Waitlist submissions are written to `data/waitlist.csv` under the project root.
- This is suitable for a single Lightsail instance with persistent disk.
- If the site later runs on multiple app instances, move waitlist storage to SQLite, RDS, or another shared datastore.

## Path structure

- Waitlist data: `data/waitlist.csv`
- NBA outputs: `Best_Bets/`, top-level CSVs such as `projections.csv`, `lines_today.csv`, and `teams_today.csv`
- MLB outputs: `mlb/outputs/` and `data/mlb/`
- Logs generated by helper scripts: `outputs/grade_log.txt` for the legacy grading helper, plus system logs from cron or `systemd`
- Future cron jobs should run from the project root with `EDGERANKED_NBA_BASE_DIR=/path/to/repo` and sibling env vars for MLB/UFC/PGA as needed

## Cron jobs to add later

Recommended home for scheduled jobs on Lightsail:

- System cron for simple VM scheduling, or
- `systemd` timers if you want better logging/restart behavior.

Suggested schedules:

- NBA refresh: 3x daily
  - Example windows: 9:00 AM ET, 1:00 PM ET, 6:00 PM ET
- MLB refresh: 3x daily
  - Example windows: 10:00 AM ET, 2:00 PM ET, 6:30 PM ET
- PGA refresh: daily or tournament-aware
  - Daily baseline: 7:00 AM ET
  - Add tournament-specific refreshes before each round lock once the PGA pipeline is finalized

## Pre-launch checklist before actual migration

- Set explicit base-dir environment variables for every sport on the Lightsail host.
- Decide whether the Render snapshot scripts should be retired or generalized.
- Confirm file permissions for CSV outputs and `data/waitlist.csv`.
- Put nginx or Caddy in front of Gunicorn for TLS termination and static request handling.
- Add process supervision for the web app and cron jobs.
